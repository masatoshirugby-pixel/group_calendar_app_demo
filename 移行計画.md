# cutieStreet_app → group_calendar_app_demo 本格移行計画

最終更新: 2026-03-25

---

## 移行の目的

cutieStreet_app は3グループ固定の個別アプリだが、
group_calendar_app_demo を**任意のアソビシステム系グループに対応できる汎用アプリ**として育て、
将来的に cutieStreet_app を廃止・統合する。

## フェーズ別計画

---

### Phase 1: バックエンド基盤構築
**目標: demo アプリが自前 DB とパイプラインを持ち、cutieStreet_app DB への依存をなくす**

#### 1-1. グループ設定の汎用化

cutieStreet_app はグループ情報がコード内に散在しているため、demo では設定ファイルで一元管理する。

```python
# groups_config.py（新規作成）
GROUPS = [
    {
        "slug": "cutiestreet",
        "name": "CUTIE STREET",
        "x_account": "CUTIE_STREET_",
        "site": "cutiestreet.asobisystem.com",
    },
    {
        "slug": "candytune",
        "name": "CANDY TUNE",
        "x_account": "CANDY_TUNE_",
        "site": "candytune.asobisystem.com",
    },
    ...
]
```

#### 1-2. DB・モデル移植

cutieStreet_app の `db.py` / `models.py` を移植し、demo 自身の PostgreSQL を持つ。
`account` カラムに slug（"cutiestreet" 等）を使用する（cutieStreet_app との混在回避）。

**Railway での対応:**
- demo 用の新規 PostgreSQL サービスを Railway に追加
- `DATABASE_URL` を demo 専用のものに差し替え

#### 1-3. パイプライン移植

以下のモジュールを cutieStreet_app から移植・汎用化する。

| ファイル | 移植元 | 変更点 |
|---|---|---|
| `web_fetcher.py` | cutieStreet_app/web_fetcher.py | グループ設定から動的に URL 生成 |
| `x_fetcher.py` | cutieStreet_app/x_fetcher.py | 変更なし（流用） |
| `event_classifier.py` | cutieStreet_app/claude_judge.py | 改名のみ |
| `event_utils.py` | cutieStreet_app/event_utils.py | 変更なし（流用） |
| `db.py` | cutieStreet_app/db.py | account を slug に統一 |
| `models.py` | cutieStreet_app/models.py | 変更なし（流用） |
| `scheduler.py` | cutieStreet_app/scheduler.py | GROUPS 設定を参照するよう汎用化 |

#### 1-4. APIエンドポイント拡充

| メソッド | パス | 説明 |
|---|---|---|
| GET | /health | ヘルスチェック（変更なし） |
| GET | /groups | グループ一覧（変更なし） |
| GET | /schedule?group={slug} | DB からイベント取得に変更 |
| POST | /fetch?group={slug} | 指定グループのフルパイプライン実行 |
| POST | /fetch/web?group={slug} | Web のみ実行（X API 不使用） |

#### 1-5. データ移行

cutieStreet_app の DB から demo の DB へ既存データをコピー。
`account` カラムを `CUTIE_STREET_` → `cutiestreet` に変換して投入。

```sql
-- 移行クエリ例
INSERT INTO events (post_id, post_text, post_url, ...)
SELECT post_id, post_text, post_url, ...
FROM cutiestreet_app.events
WHERE account = 'CUTIE_STREET_';
```

#### Phase 1 完了条件
- demo が自前 DB を持ち、cutieStreet_app DB への依存がゼロ
- 毎日のパイプラインが demo バックエンドで動作している
- cutiestreet / candytune / sweetsteady の3グループでデータが表示される

### Phase 2: フロントエンド機能移植
**目標: cutieStreet_app の高機能フロントエンドを demo に移植し、多グループ対応させる**

#### 2-1. カテゴリフィルター

- カテゴリを複数選択してカレンダーの表示を絞り込む
- cutieStreet_app の実装を移植し、グループ固定のカラーをなくす

#### 2-2. 申込締切の紐づけ表示

- 親イベントクリック → 右パネルに申込締切一覧を表示
- `post_id` の前方一致（`{親ID}_deadline`）で紐づけ
- 申込締切クリック → 親イベント情報を右パネルに表示
- 紐づくバッジをリングでハイライト

#### 2-3. 詳細パネルの情報拡充

| 項目 | 現状 | Phase 2 |
|---|---|---|
| カテゴリ | ✅ | ✅ |
| 開催日 | ✅ | ✅ |
| 本文 | ✅ | ✅ |
| ソースリンク | ✅（公式のみ） | ✅（X / 公式を判別して表示） |
| 会場 | ❌ | ✅ |
| 画像 | ❌ | ✅ |
| 申込締切一覧 | ❌ | ✅ |
| 先着順バッジ | ❌ | ✅ |
| 投稿日時 | ❌ | ✅（source = 'x' の場合のみ） |

#### 2-4. 日程未確定イベントの展開表示

- カレンダー下部に「▶ 日程未確定 N件」を表示
- クリックで展開し、一覧表示

#### Phase 2 完了条件
- demo のフロントエンドが cutieStreet_app と同等の機能を持つ
- 多グループ対応（グループが変わっても同じ UI で動作）

### Phase 3: メール統合（オプション）
**目標: Gmail 申込締切情報を demo に統合する**

> cutieStreet_app の `email_fetcher.py` をそのまま移植。
> 接続する Gmail アカウントは cutieStreet_app と同じものを使用。

#### 3-1. 移植内容

| ファイル | 移植元 | 変更点 |
|---|---|---|
| `email_fetcher.py` | cutieStreet_app/email_fetcher.py | 変更なし（流用） |

#### 3-2. 追加エンドポイント

| メソッド | パス | 説明 |
|---|---|---|
| GET | /emails | メール一覧取得 |

#### 3-3. フロントエンド追加

- EmailPanel コンポーネントを移植
- 当選通知バナー（WinBanner）を移植

#### 3-4. 環境変数追加

```env
GMAIL_ADDRESS=xxxx
GMAIL_APP_PASSWORD=xxxx
```

#### Phase 3 完了条件
- demo で Gmail 申込締切情報がカレンダーに表示される

---

### Phase 4: カットオーバー
**目標: cutieStreet_app を廃止し、demo に完全移行する**

1. demo の URL を cutieStreet_app と同等のものに変更（または Vercel のドメイン設定で対応）
2. cutieStreet_app の Railway サービス・Vercel プロジェクトを停止
3. cutieStreet_app の DB を削除（demo DB にデータが引き継がれていることを確認後）

## 推奨実装順序

```
Phase 1（約1〜2週間）
  └─ 1-1 グループ設定汎用化
  └─ 1-2 DB・モデル移植
  └─ 1-3 パイプライン移植（Web → X → スケジューラー）
  └─ 1-4 APIエンドポイント追加
  └─ 1-5 データ移行

Phase 2（約1週間）
  └─ 2-1 カテゴリフィルター
  └─ 2-2 申込締切の紐づけ表示
  └─ 2-3 詳細パネル拡充
  └─ 2-4 日程未確定の展開表示

Phase 3（約3〜5日）
  └─ メール統合

Phase 4（1日）
  └─ カットオーバー・旧サービス停止
```
