# グループカレンダー デモアプリ 開発方針

最終更新: 2026-03-25

---

## 1. アプリ概要

アソビシステム系アイドルグループのスケジュールをカレンダー形式で表示するデモアプリ。
cutieStreet_app（本番アプリ）のSaaS横展開を想定したプロトタイプ。

- **CUTIE STREET / CANDY TUNE / SWEET STEADY**: cutieStreet_app が蓄積した PostgreSQL のデータをそのまま参照
- **その他6グループ**: オンデマンドで公式サイトをスクレイピング

---

## 2. 技術スタック

| レイヤー | 技術 | 備考 |
|---|---|---|
| フロントエンド | Next.js 14 App Router | cache: no-store（常に最新取得） |
| フロントホスティング | Vercel | GitHub 連携で自動デプロイ |
| バックエンド | Python + FastAPI | DB なし・オンデマンド取得 |
| バックホスティング | Railway | |
| データベース | PostgreSQL（cutieStreet_app 共有） | cutiestreet / candytune / sweetsteady のみ |
| Webスクレイピング | requests + BeautifulSoup | その他グループのスケジュール取得 |

## 4. 開発ルール

### データ取得方針
- **自前DBなし**。cutieStreet_app の DB を読み取り専用で参照するか、その場でスクレイピング
- X API・Gmail・パイプライン処理は本アプリの責務外（cutieStreet_app に委任）

### コスト管理
- X API は本アプリからは呼び出さない（cutieStreet_app の DB に蓄積済みデータを使う）

### デプロイフロー
```
git push → GitHub → Vercel 自動デプロイ（フロント）
                  → Railway 自動デプロイ（バック）
```

### ブランチ運用
- `main`: 本番
